# Next.js SaaS Render Starter

Generated by `stripe build` as the first supported vertical slice.

## What this starter gives you

- Next.js App Router app
- Bare-bones Stripe Checkout subscription flow for $10.00 / month
- Terraform-managed Stripe product and recurring price in `infra/stripe/main.tf`
- Optional Clerk auth wiring with checkout protection
- Optional Render Postgres wiring with starter `users` and `subscriptions` tables
- Optional Vercel hosting metadata


## Verify the starter

Open the landing page and use it as the setup summary for the generated app.

- `Subscribe` opens the starter subscription flow
- `View Database` opens the same starter database summary modal used in the integration panel
- `View sign in` verifies the generated Clerk entry points, then sends signed-in users to `/dashboard`
- `View Database` shows the starter Render Postgres connection state plus row counts for `users` and `subscriptions`
- `npm run deploy` uses the pulled Vercel credentials to publish the app


## Local development

1. Install dependencies:
   `npm install`
1. The generator attempts to create the Stripe catalog resources for you with Terraform.
1. Verify that `STRIPE_PRODUCT_ID`, `STRIPE_PRICE_ID`, and `STRIPE_SECRET_KEY` are present in `.env.local`.
1. When Render Postgres is configured, the starter automatically creates its `users` and `subscriptions` tables the first time a database-backed route runs.
1. When Clerk is enabled, direct sign-ins and sign-ups flow through `/auth/sync`, and signed-in checkouts also upsert the app user before creating the Stripe Checkout Session.
1. To enable Stripe webhook syncing after the app has a public URL, run:
   - `cd infra/stripe`
   - `export TF_VAR_stripe_api_key=sk_test_...`
   - `export TF_VAR_webhook_endpoint_url=https://your-app.example.com/api/webhooks/stripe`
   - `terraform init`
   - `terraform apply`
   - `terraform output -raw webhook_secret`
1. Copy the Terraform `webhook_secret` output into `STRIPE_WEBHOOK_SECRET` in `.env.local` or your deployment environment once your webhook endpoint URL is configured.

1. `npm run deploy` publishes to Vercel when these pulled env vars are available: `VERCEL_TOKEN`, `VERCEL_PROJECT_ID`, `VERCEL_TEAM_ID`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_URL`, `VERCEL_URL`.

1. If needed, copy the Terraform `product_id` and `price_id` outputs into `STRIPE_PRODUCT_ID` and `STRIPE_PRICE_ID` in `.env.local`.
1. If the generator did not finish provisioning your add-ons, run:
   - `stripe plugin install projects`
   - `stripe projects init`
   - `stripe projects add vercel/project`
   - `stripe projects add render/postgres`
   - `stripe projects add clerk/auth`
   - `stripe projects env --pull`
1. Start the app:
   `npm run dev`

## Routes

- `/` landing page
- `/dashboard` signed-in starter workspace

- `/checkout` starter subscription checkout flow
- `/success` checkout success return page
- `/cancel` checkout cancel return page
- `/api/health` runtime configuration summary
- `/auth/sync` best-effort Clerk-to-database sync route
- `/api/billing-portal` creates Stripe customer portal sessions for signed-in subscribed users
- `/api/db` database summary route with starter table counts when Render Postgres is provisioned
- `/api/webhooks/stripe` Stripe webhook endpoint for initial subscription linking once `STRIPE_WEBHOOK_SECRET` is configured

- `claude "Help me turn this into a real product. Follow prompts/starter-to-product.md."`
- `agent "Help me turn this into a real product. Follow prompts/starter-to-product.md."`

If you are using another AI coding tool, open `prompts/starter-to-product.md` and paste it into the assistant.

The prompt tells the agent to read `AGENTS.md`, inspect the starter, ask one discovery question at a time when the brief is still fuzzy, confirm the product name with the user, work in phases with the landing page first, remove starter status UI from the product experience, and only then implement without breaking the existing Stripe, auth, database, and Terraform wiring. It also points the agent at a frontend design reference to avoid generic AI-starter styling.
